# Вольтаж — Android-клиент

Нативное Android-приложение для автоматизированного подбора, сравнения цен и проверки совместимости комплектующих ПК. Клиентская часть программного комплекса «Вольтаж».

Приложение работает в связке с серверной частью (FastAPI + PostgreSQL, см. репозиторий сервера): получает каталог товаров с трёх российских ритейлеров (DNS, «Ситилинк», «Регард»), позволяет собрать конфигурацию ПК и проверить её совместимость, а также сохраняет сборки локально для работы без интернета.

Выпускная квалификационная работа, направление 09.03.02 «Информационные системы и технологии».

## Возможности

- Просмотр каталога комплектующих по 8 категориям с ценами сразу трёх магазинов
- Сборка конфигурации: добавление компонентов, выбор магазина с лучшей ценой для каждой позиции
- Проверка технической совместимости сборки через серверный API
- Сохранение сборок и история с возможностью редактирования
- Автономная работа без интернета — текущая и сохранённые сборки хранятся локально

## Экраны приложения

Три основных раздела доступны через нижнюю навигацию: **Каталог** — просмотр категорий комплектующих; **Сборка** — текущая конфигурация с ценами и статусом совместимости; **История** — список ранее сохранённых сборок с возможностью редактирования.

## Подключение к серверу

Адрес сервера не хранится в коде. Он задаётся переменной `SERVER_BASE_URL` в `local.properties` и пробрасывается в приложение на этапе сборки через `BuildConfig.BASE_URL` (см. `app/build.gradle.kts`). Это значение использует `RetrofitClient` в `ApiService.kt`.

Если `SERVER_BASE_URL` не задан, при сборке подставляется заглушка `http://localhost/`, и сетевые запросы работать не будут — без `local.properties` приложение не сможет подключиться к серверу (см. шаг 2 ниже).

Для локальной разработки достаточно указать в `local.properties` адрес локального сервера вместо продуктивного VPS. Если используется HTTP (не HTTPS), также нужно обновить `network_security_config.xml`, чтобы Android разрешил подключение по используемому протоколу и адресу.

## Сборка проекта

1. Открыть проект в Android Studio.
2. Создать (или дополнить) файл `local.properties` в корне проекта и указать адрес сервера:
   ```properties
   SERVER_BASE_URL=http://<адрес-сервера>:8000/
   ```
   Файл специфичен для конкретной машины и не должен попадать в репозиторий — он указан в `.gitignore`.
3. Дождаться синхронизации Gradle (путь к Android SDK в `local.properties` добавляется автоматически).
4. Запустить на устройстве/эмуляторе с Android 10.0+ (API 29).
